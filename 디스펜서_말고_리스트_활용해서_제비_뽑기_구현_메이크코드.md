## 소개
이 영상은 마인크래프트 에듀케이션 에디션의 메이크코드를 활용하여 문자열 리스트를 사용한 제비 뽑기 시스템을 구현하는 방법을 소개합니다. 기존의 디스펜서를 이용한 제비 뽑기 방식과 달리, 코드를 통해 더 효율적이고 유연한 제비 뽑기 시스템을 만드는 방법을 자세히 설명합니다. 해리포터의 기숙사 이름(그리핀도르, 슬리데린, 래번클로, 후플푸프)을 예시로 활용하여 중복 없는 랜덤 추첨 시스템을 구현합니다.

## 주요 내용
### 1. 문자열 리스트 활용 원리
- 문자열 리스트(String List)와 일반 리스트의 차이점
- 리스트 내 데이터 저장 및 관리 방법
- 메이크코드에서 문자열 리스트 생성 및 초기화 방법

### 2. 랜덤 추첨 알고리즘 구현
- 랜덤 인덱스 생성을 통한 리스트 내 항목 선택
- 뽑은 항목을 리스트에서 제거하여 중복 방지
- 리스트 길이에 따른 동적 랜덤 범위 설정

### 3. 중복 없는 제비 뽑기 시스템
- 한 번 뽑은 항목이 다시 등장하지 않도록 설계
- 리스트 길이 변화에 따른 랜덤 범위 자동 조정
- 모든 항목이 뽑힐 때까지 반복 가능한 구조

### 4. 메이크코드 고급 기능 활용
- `remove value at` 명령어를 통한 리스트 요소 제거
- 리스트 길이 속성을 활용한 동적 범위 설정
- 채팅 명령어와 연동한 사용자 인터페이스 구현

## 자주 묻는 질문
**Q: 일반 리스트와 문자열 리스트의 차이점은 무엇인가요?**
A: 일반 리스트는 숫자 데이터를 저장하는 반면, 문자열 리스트는 텍스트 데이터를 저장합니다. 텍스트를 다루는 제비 뽑기의 경우 문자열 리스트를 사용해야 합니다. 메이크코드에서는 '변수' 메뉴의 '고급' 탭에서 찾을 수 있습니다.

**Q: 리스트의 인덱스가 왜 0부터 시작하나요?**
A: 프로그래밍에서 배열이나 리스트의 인덱스는 일반적으로 0부터 시작합니다. 이는 컴퓨터 메모리 주소 계산 방식에서 유래된 것으로, 대부분의 프로그래밍 언어가 이 규칙을 따릅니다. 따라서 리스트의 첫 번째 요소는 인덱스 0, 두 번째 요소는 인덱스 1이 됩니다.

**Q: 디스펜서 대신 리스트를 사용하는 장점은 무엇인가요?**
A: 디스펜서는 실제 아이템을 배치해야 하고, 최대 9개의 슬롯 제한이 있으며, 물리적인 위치가 필요합니다. 반면 리스트를 활용한 제비 뽑기는 개수 제한이 없고, 코드 한 번으로 다양한 장소에서 실행할 수 있으며, 쉽게 아이템을 추가하거나 수정할 수 있는 장점이 있습니다.

**Q: 뽑은 항목을 다시 리스트에 추가하는 방법은 어떻게 되나요?**
A: 현재 코드는 중복 없는 제비 뽑기를 구현하기 위해 뽑은 항목을 제거합니다. 만약 뽑은 후 다시 넣고 싶다면, `remove value at` 명령어를 사용하지 않거나, 뽑은 후 다시 `add value to list` 명령어로 추가하는 코드를 넣으면 됩니다.
